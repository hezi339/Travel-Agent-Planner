## 一、项目背景与目标

### 1.1 市场背景

根据小红书桌面调研，自由行用户存在三大核心痛点：

- **行程规划焦虑**：攻略耗时耗力、信息过载，「做攻略比旅行本身还累」
- **穿搭焦虑**：次抛衣服浪费、出片压力大、穿搭与场景不匹配
- **同行协调成本高**：多人旅行规划意见难以统一

其中，**穿搭焦虑**是小红书平台特有的强需求痛点。「穿搭推荐 + 场景匹配」具备高用户价值，且是竞品未充分覆盖的差异化机会。

### 1.2 产品定位

**一句话定位**：基于 AI 视觉识别与多源数据融合的智能旅行穿搭规划助手。

**核心价值主张**：上传一张旅行灵感截图，AI 为你生成完整的「行程 + 穿搭」攻略——从攻略灵感识别到天气适配穿搭，一站式解决「穿什么、怎么玩」的旅行决策焦虑。

### 1.3 产品目标

| 目标维度 | 具体目标 |
|----------|----------|
| 用户价值 | 将旅行规划时间从平均 3–5 小时缩短至 5 分钟以内 |
| 商业目标 | 首月 DAU 1000+，7 日留存率 ≥ 30% |
| 技术目标 | 图片识别准确率 ≥ 85%，穿搭推荐用户满意度 ≥ 4.0/5.0 |

## 二、用户画像与场景

### 2.1 核心用户画像

**主要用户：「精致旅行族」小 A**

- 22–30 岁，一二线城市白领 / 学生
- 小红书重度用户，旅行前收藏大量攻略与穿搭灵感
- 追求「出片率」，预算敏感，有穿搭焦虑
- **痛点**：收藏 100+ 篇攻略不知如何整合；衣服只穿一次；到了当地发现穿搭不适合天气

**次要用户：「效率型旅行者」小 B**

- 25–35 岁，商务人士 / 自由职业者
- 时间宝贵，希望快速获得可执行方案
- **痛点**：不想花大量时间做攻略，又怕踩坑

### 2.2 核心使用场景

| 场景 | 描述 |
|------|------|
| **攻略灵感转执行**（主场景） | 上传小红书攻略截图 → 5 分钟内获得行程 + 穿搭 + 天气提醒 |
| **穿搭灵感复用** | 上传穿搭图 + 目的地 → AI 跨场景适配并推荐相似款 / 购买链接 |
| **多人协作规划** | 多人上传偏好 → AI 整合生成兼顾双方的行程与穿搭方案 |

## 三、产品架构

### 3.1 目标系统架构（完整愿景）

```
┌─────────────────────────────────────────────────────────────┐
│                   用户交互层（微信小程序）                    │
│   图片上传 · 文本输入 · 行程查看 · 穿搭预览                  │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│                     Agent 编排层                             │
│   智谱 AI Agent：意图识别 · 任务分解 · 结果整合 · 风格生成    │
└─────────────────────────────────────────────────────────────┘
          │                   │                   │
┌─────────▼──────┐  ┌─────────▼──────┐  ┌─────────▼──────┐
│  视觉识别模块   │  │  地理数据模块   │  │  知识库模块     │
│ OCR/场景/风格  │  │ 高德 POI/天气  │  │ RAG/用户偏好   │
└────────────────┘  └────────────────┘  └────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│  商业化模块：淘宝/京东/拼多多联盟 · 智谱 CogView 穿搭效果图    │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 数据流转（摘要）

1. 用户上传截图 → 水印处理 → Vision 识别 → 结构化存入 RAG
2. 用户输入需求 → Agent 分解 → 高德地理编码 + 天气 API
3. LLM 整合信息 → 生成「行程 + 穿搭」→ CogView 效果图
4. 电商联盟商品推荐 → 用户点击 → 佣金收益

## 四、功能需求

### M1 · 视觉灵感识别（Vision Input Module）

**功能**：上传社交媒体截图，自动识别攻略与穿搭关键信息。

**处理流程**：

1. 图片预处理：格式转换、压缩、水印检测与模糊
2. 多模态识别（智谱 Vision / Embedding）：
   - OCR 提取景点、店铺、行程文字
   - 场景识别（古建筑 / 海滩 / 都市等）
   - 穿搭风格识别（日系 / 韩系 / 法式 / 新中式等）
   - 色彩分析：主色调与配色方案
3. 结构化 JSON 存入 RAG 向量库

**技术要求**：JPG/PNG/WebP，单张 ≤ 10MB，识别 ≤ 3 秒，支持批量最多 5 张。

**Demo 实现状态**：🚧 Vision 结构化 JSON 入库已实现；📋 水印 / 色彩分析未实现。

---

### M2 · 智能行程规划（Itinerary Planning Module）

**功能**：基于 RAG 攻略与用户文本需求，生成弹性分日行程。

**处理流程**：

1. Agent 解析意图（目的地、天数、偏好）
2. RAG 检索相关攻略
3. 高德地理编码 → 经纬度
4. 路径优化、时间规划、弹性可选项设计

**Demo 实现状态**：🚧 基础分日行程（LLM + RAG）；📋 路径规划 / 地图标注未实现。

---

### M3 · 智能天气适配穿搭（Weather-Adaptive Outfit Module）

**功能**：结合实时 / 预报天气与城市风格库，生成个性化穿搭建议。

**处理流程**：

1. 高德 adcode + 经纬度
2. 天气 API（温度、降雨、UV、湿度）
3. 匹配城市风格库（首期 8 城：广州、苏州、三亚、成都、西安等）
4. 生成每日穿搭：上装 / 下装 / 鞋履 / 配饰 / 出片场景 / 天气适配提示

**Demo 实现状态**：🚧 规则工具 `recommend_clothing`；📋 城市风格库融合未实现。

---

### M4 · AI 穿搭效果图（AI Outfit Visualization Module）

**功能**：基于穿搭方案，用文生图生成虚拟穿搭效果图。

**技术方案**：

- 主：智谱 CogView；备：Stable Diffusion / Midjourney
- 输出 512×768 或 768×1024，支持「换一套」、保存分享
- 成本控制：热门城市优先，每用户每日限 3 张，相同描述缓存复用

**Demo 实现状态**：📋 规划中。

---

### M5 · 电商导购与佣金变现（E-commerce Monetization）

**功能**：穿搭方案中嵌入可购买商品链接，获取联盟佣金。

**接入**：淘宝联盟（优先）、京东联盟、拼多多联盟。

**匹配逻辑**：LLM 提取单品关键词 → 联盟搜索 → 按销量 / 评分 / 价格过滤 → 追踪短链。

**Demo 实现状态**：📋 规划中。

---

### M6 · RAG 知识库管理（Knowledge Base Module）

**功能**：本地向量库存储攻略与穿搭信息，支持语义检索。

**技术方案**：

- 向量库：ChromaDB / Milvus / FAISS
- Embedding：智谱 embedding 系列
- 检索：Dense + Hybrid Search + Rerank

**Demo 实现状态**：🚧 Chroma + embedding-3 + Top-K 检索；📋 Hybrid / Rerank / 去重未实现。

---

### M7 · 多源 API 集成网关（API Gateway）

**功能**：统一鉴权、限流、预算监控、容错降级。

**要求**：月度 API 预算上限 500 元，超限触发降级策略。

**Demo 实现状态**：📋 规划中。

---

### M8 · 穿搭灵感复用（Outfit Inspiration Reuse）

**功能**：上传穿搭参考图 + 目的地，跨场景风格适配与商品推荐。

**Demo 实现状态**：📋 规划中。

---

### M9 · 多人协作规划（Collaborative Planning）

**功能**：旅行房间、偏好整合、行程投票。

**Demo 实现状态**：📋 规划中。

---

### M10 · 穿搭日历与行李清单（Packing List Module）

**功能**：场景化装备清单、勾选打包、PDF / 长图分享。

**Demo 实现状态**：🚧 `checklist_categories` 分类清单；📋 勾选 / PDF / 分享未实现。

## 五、非功能需求（摘要）

| 指标 | 目标值 |
|------|--------|
| 图片识别 | ≤ 3 秒 |
| 行程生成 | ≤ 10 秒 |
| 穿搭方案 | ≤ 8 秒 |
| AI 效果图 | ≤ 15 秒 |
| 并发用户 | ≥ 100 |
| 系统可用性 | ≥ 99.5% |

**安全与隐私**：HTTPS 传输；Demo 阶段原图保留 7 天；正式运营仅存结构化 JSON；API Key 服务端存储；微信小程序隐私合规。

**平台**：首期微信小程序；微信基础库 2.19.4+；iOS 12+ / Android 8+。
